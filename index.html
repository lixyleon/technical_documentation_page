Introduccion

MySQL es un sistema de administración de bases de datos (Database Management System, DBMS) para bases de datos relacionales open source. Se sustenta en SQL: Structure Query Language (Lenguaje de Consulta Estructurado).

MySQL utiliza múltiples tablas para almacenar y organizar la información. Fue escrito en C y C++ y destaca por su gran adaptación a diferentes entornos de desarrollo, permitiendo la interacción con los lenguajes de programación de más uso como PHP, Perl y Java; así como, su integración en distintos sistemas operativos.

Conocimientos previos

Para comenzar a trabajar con una base de datos debes conocer su definición: Una base de datos es un gran contenedor de información almacenada de forma organizada. Los datos estan organizados de forma tal que no exista redundacia y se puedan establecer relaciones entre ellos.

La base de datos contiene las tablas identificadas con un nombre único donde se uardarán los datos, los cuales también son identificados de forma única.

Crear y eliminar bases de datos

Crear bases de datos

Para crear una base de datos a través de sentencias de SQL, se utiliza la siguiente sintaxis:

CREATE DATABASE farmacia;

Con la sentencia anterior, se ha creado una base de datos denominada farmacia. Es importante señalar que las palabras “CREATE” y “DATABASE” están escritas en mayúsculas, visto que corresponden a palabras reservadas de SQL y deben diferenciar de las palabras utilizadas para nombrar el resto de los elementos. Al finalizar cada instrucción debe escribirse “;” con lo cual se garantiza el fin de la instrucción.

Eliminar bases de datos

Para eliminar una base de datos se utiliza la instrucción DROP. Por ejemplo, para eliminar la base de datos creada en el punto anterior, ejecutamos la siguiente sentencia:

DROP DATABASE farmacia;

Crear modificar y eliminar tablas

Crear tabla

Una tabla es una estructura que organiza los datos en columnas y filas, donde cada columna es un campo y cada fila un registro. Cada intersección entre una columna y una fila, contiene un dato específico, es decir, un solo valor.

Cada campo debe tener un nombre, el cual debe hacer referencia al tipo de dato que almacena, a los fines de acilitar la interpretación de los datos que contiene la tabla.

Nombre del producto Cantidad disponible Precio Codigo
Ibuprofeno 500 25.40 87963
Atamel 700 30.80 58763
Naproxeno 1700 50.50 58863

Gráficamente la tabla productos contiene tres campos: nombre del producto, cantidad disponible y precio. Luego tenemos dos registros almacenados en esta tabla:

  • El primer registro contiene para el primer campo (nombre del producto) el valor Ibuprofeno, para el segundo campo (cantidad disponible) 500, el tercer campo (precio) 25.40 y el cuarto (código) 87963
  • El segundo registro contiene para el primer campo el valor Atamel, para el segundo campo (cantidad disponible) 700, el tercer campo (precio) 30.80 y el cuarto (código) 58763
  • El tercer registro contiene para el primer campo el valor Naproxeno, para el segundo campo (cantidad disponible) 1700, el tercer campo (precio) 50.50 y el cuarto (código) 58863.

Al crear una tabla debemos resolver qué campos (columnas) tendrá y que tipo de datos almacenarán cada uno de ellos, es decir, su estructura. En ese sentido, para crear la tabla con las características antes descritas, ejecutaremos la siguiente sentencia:

CREATE TABLE producto (
nombre_producto VARCHAR (50),
cantidad_disponible INT,
precio FLOAT
, codigo VARCHAR (20)
);

Modificar tablas

La instrucción ALTER TABLE permite modificar la estructura de una tabla existente. Mediante esta instrucción podemos agregar o eliminar columnas, modificar el tipo de columna (tipo de dato), renombrar campos y la tabla. Aplicaremos a manera de ejemplo ciertos cambios a la tabla productos.

Renombrar columna: se cambia el nombre del campo nombre_producto por nm_prod

ALTER TABLE producto CHANGE nombre_producto nm_prod VARCHAR (50);

Eliminar columna: se elimina el campo codigo

ALTER TABLE producto DROP COLUMN codigo;

Agregar una nueva columna

ALTER TABLE producto ADD descripcion VARCHAR(100);

Tipos de datos

Como se indicó en puntos anteriores, al crear una tabla es necesario definir su estructura, es decir, los campos (columnas) que tendrá y el tipo de datos almacenará cada uno de ellos. A continuación se listan algunos tipos de datos básicos:

  • VARCHAR: se usa para almacenar cadenas de caracteres. Una cadena es una secuencia de caracteres que se colocan entre comillas (simples). Adicionalmente, es necesario definir su longitud con un máximo de 255 caracteres. Por ejemplo, para almacenar cadenas de hasta 10 caracteres, definimos un campo de tipo VARCHAR(10). Si asignamos una cadena de caracteres de mayor longitud que la definida, la cadena se corta. Por ejemplo, si definimos un campo de tipo VARCHAR(10) y le asignamos la cadena 'Buenos dias sra Laura', se almacenará 'Buenos dia'.
  • INTEGER: se usa para guardar valores numéricos enteros, de -2000000000 a 2000000000 aproximadamente. Se utiliza cuando se requiere almacenar cantidades, número de identificación, otros.
  • FLOAT: se utiliza para almacenar valores numéricos con decimales. Como separador de la porción decimal se coloca el punto (.). Definimos campos de este tipo para precios, temperatura, peso, otros.
Insertar modificar y eliminar registros en una tabla

Insertar registros

Un registro es una fila de la tabla que contiene los datos. Cada registro tiene un dato por cada columna. Al insertar registros en una tabla debe tener en cuenta la cantidad y el orden de los campos. Para insertar un registro en la tabla productos deberá aplicar la siguiente sentencia:

Recuerde que la tabla productos del ejemplo anterior ha sido modificada, en su actual estructura los campos son: nm_prod, cantidad_disponible, precio y descripcion

INSERT INTO producto (nm_prod, cantidad_disponible, precio, descripcion) VALUES
('Pantoprazol',1000, 25.8, protector gastrico);

Iniciamos la instrucción con “INSERT INTO” luego especificamos los nombres de los campos entre paréntesis y separados por comas y posteriormente los valores para cada campo (también entre paréntesis y separados por comas).

El comando "SELECT" recupera los registros de una tabla. Luego del comando SELECT indicamos los nombres de los campos a recuperar, generando una salida con los registros y los valores que contiene cada registro en cada campo de la tabla.

SELECT nm_prod, cantidad_disponible, precio, descripcion FROM productos;

Modificar registros

Para modificar uno o varios datos de uno o varios registros utilizamos "UPDATE". Por ejemplo, en la tabla productos se requiere modificar el campo descripción para el producto Pantoprazol, para lo cual se ejecutará la siguiente sentencia:

UPDATE productos descripcion = 'Protector' WHERE nm_prod = 'Pantoprazol';

Note que se utilizó la instrucción WHERE, de no ser incluida se modificaría la descripción de todos los registros de la tabla

Eliminar registros

Para eliminar los registros de una tabla usamos el comando "DELETE":

DELETE FROM productos;

Con la sentencia anterior, serán eliminados todos los registros de la tabla. Si se requiere eliminar un registro deberá especificarse con la instrucción WHERE:

DELETE FROM productos WHERE nm_prod = 'Ibuprofeno';

Clave primaria y clave foranea

Clave primaria

Una clave primaria es un campo (o varios) que identifica un solo registro (fila) en una tabla. Los valores de ese campo no se repiten ni pueden ser nulos, por tanto, es posible identificar un registro a través de dicho campo.

Por ejemplo, si tenemos una tabla con datos de clientes, el número de identificación puede establecerse como clave primaria, visto que es un valor que no se repite, pudiendo haber coincidencias en tros campos como nombre, edad, ingreso, entre otros.

Al momento de creación de la tabla, establecemos que un campo sea clave primaria de la siguiente forma:

CREATE TABLE empleados (
id INT PRIMARY KEY,
nombre VARCHAR(40),
apellido VARCHAR(40),
num_identificacion INT,
telefono VARCHAR(20)
);

Campo con autoincremento

Un campo de tipo entero puede tener el atributo 'AUTO_INCREMENT'. Los valores de un campo 'AUTO_INCREMENT', se inician en 1 y se incrementan en 1 de forma automáticamente, con la creación de un nuevo registro. Por lo general, es utilizado para generar valores únicos para cada nuevo registro que se inserta en una tabla. Sólo puede haber un campo "AUTO_INCREMENT" y debe ser clave primaria. La forma de definir un campo con el atributo 'AUTO_INCREMENT' es la siguiente:

CREATE TABLE empleados (
id AUTO_INCREMENT INT PRIMARY KEY,
nombre VARCHAR(40),
apellido VARCHAR(40),
num_identificacion INT,
telefono VARCHAR(20)
);

Clave foránea

Las claves foráneas son referencias a registros de otras tablas estableciendo una relación entre ellas. La clave foránea apunta siempre a la clave primaria de la tabla a la cual hace referencia.

A los efectos de ilustrar el uso de un campo definido como clave foránea, crearemos la tabla sucursales:

CREATE TABLE sucursales (
id INT AUTO_INCREMENT PRIMARY KEY,
nombre VARCHAR(40),
direccion VARCHAR(50),
telefono VARCHAR (20)
);

Crearemos la tabla empledados donde asociamos a cada empleado a una sucursal

CREATE TABLE empleados (
id INT PRIMARY KEY,
nombre VARCHAR(40),
apellido VARCHAR(40),
num_identificacion INT,
telefono VARCHAR(20),
id_sucursal INT,
FOREIGN KEY (id_sucursal) REFERENCES sucursales(id)
);

Columnas calculadas

Es posible obtener salidas en las cuales una columna sea el resultado de un cálculo y no un campo de una tabla. Por ejemplo, creamos tabla inventario:

CREATE TABLE inventario (
id INT PRIMARY KEY,
nombre VARCHAR(40),
precio FLOAT,
cantidad INT,
fecha_vencimiento DATE
);

Ahora, luego de ingresar datos a dicha tabla, suponga que para cada producto se requiere conocer el monto que se obtendría al vender la totalidad de cada producto al precio fijado, obtenido por la multiplación de la cantidad por el precio. Esto puede obtenerse de la siguiente forma:

SELECT id, nombre, precio ,cantidad, precio*cantidad FROM inventario ;

Clausula ORDER del SELECT

Podemos ordenar el resultado de un "SELECT" para que los registros se muestren ordenados por algún campo, para ello usamos la cláusula "ORDER". Por ejemplo, recuperamos los registros de la tabla inventario ordenados por el nombre:

SELECT id, nombre, precio,cantidad FROM inventario ORDER BY nombre ;

El resultado es una tabla con los campos especificados, de la tabla inventario, ordenados por el nombre del producto. Por defecto, los ordena de manera ascendente, pero podemos ordenarlos de forma descendente, con el uso de la palabra clave "DESC"

SELECT id, nombre, precio,cantidad FROM inventario ORDER BY nombre DESC;

Operadores logicos y relacionales

Los operadores lógicos se usan para combinar condiciones. Por ejemplo, requerimos recuperar los registros de la tabla inventarios que cumplan dos condiciones: el nombre sea igual a protector y la cantidad sea mayor o igual a 100

SELECT * inventario WHERE nombre = 'protector' AND cantidad >= 100;
